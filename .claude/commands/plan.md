# Plan Feature Command

Generate a comprehensive planning document for implementing a new feature in the Chessle codebase.

I'll analyze your feature request: $ARGUMENTS

First, let me examine the current codebase structure and existing patterns to understand how your feature should integrate with the existing architecture.

I'll then generate a detailed implementation plan that breaks down your feature into:

- Clear objectives and scope
- Key technical components required
- Step-by-step implementation checklist
- Potential challenges and questions to resolve

The plan will be saved as a markdown file in the `/agents` directory following the naming convention `[YYYY-MM-DD]-[feature-name].md`.

The document will follow the established format:

- **Objective**: Clear statement of the feature's purpose and goals
- **Key Components**: Major parts including UI components, API endpoints, database changes
- **Implementation Steps**: Detailed, actionable checklist of required tasks
- **Questions & Blockers**: Open questions and potential challenges to address

This document serves as a collaborative planning tool that can be refined during implementation, with items checked off as they're completed.
